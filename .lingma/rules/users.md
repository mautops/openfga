# 以下为 "用户规则" 重要原则，必须严格优先遵守!

## 重要原则

1. 请保持对话语言为中文;
2. 我的系统为 Mac os;
3. 请在生成代码时添加适当注释;
4. 不要执行如下命令: `pnpm dev`, `npm run dev`
5. 前端项目使用 Nextjs 16 框架 及 pnpm 来管理依赖;
6. Python 项目使用 uv 管理依赖, Python 解析器路径下: `./.venv/bin/python`
7. Golang 项目常用包: gin, gorm, viper, godotenv, zap, cobra
8. 及时写入文件，避免一次输出太多内容，导致输出报错；

## 规划与分阶段

将复杂的工作分解为若干个阶段, 在项目中 temp 目录下创建 Markdown 文件记录, 随着进展更新状态, 所有阶段完成后删除文件。阶段项格式如下:

```text
```
## 阶段 N: [Name]

**目标**: [具体可交付成果]
**成功标准**: [可测试的结果]
**状态**: [未开始|进行中|已完成]
```
```


## 决策框架

当存在多种有效方法时，基于以下几点进行选择：

1. 可测试性 - 该方案是否易于测试？
2. 可读性 - 6 个月后是否有人能看懂？
3. 一致性 - 是否符合项目现有模式？
4. 简单性 - 是否是可行的最简单方案？
5. 可逆性 - 以后修改的难度有多大？

## 组件使用

1. 尽量使用现有的组件
2. 在没有组件时，只推荐组件，禁止自动安装

## 错误处理

1. 快速失败并提供描述性信息
2. 包含用于调试的上下文
3. 在适当的层级处理错误
4. 绝不静默地吞掉异常
5. 修复错误时，使用`第一性原理` 思考，可以提高修复成功率；
